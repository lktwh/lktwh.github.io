# 函数

实际参数和形式参数

实际参数是出现在函数调用时圆括号中的表达式。形式参数是函数定义的函数头中声明的变量。

函数原型

函数头以分号结尾，就构成了函数原型，目的是告诉编译器函数长什么样子。

类型不匹配

调用函数时给的参数值与参数的类型不匹配，但是编译器会将类型转换好，很可能不是自己所想要的。

C 语言在调用函数时，永远只能传值给函数。

本地（局部）变量

  • 定义在块内的
    • 定义在函数块内
    • 定义在语句块内
    • 也可以是随便拉一对大括号来定义变量
  • 程序运行进入这个块之前，其中的变量不存在，离开这个块，其中的变量就消失了
  • 块外面定义的变量在块里面仍然有效
  • 块里面定义了和块外面同名的变量则覆盖了外面的
  • 不能在一个块内定义同名的变量
  • 本地变量不会被默认初始化
  • 参数在进入函数的时候就被初始化了
  • 返回本地变量的地址是危险的，因为本地变量是存放在栈区的，栈区的变量在函数结束时会自动释放。

全局变量

  • 定义在函数外面的变量是全局变量
  • 全局变量有全局的生存期和作用域
    • 它们与任何函数都无关
    • 在任何函数内部都可以使用它们
  • 没有做初始化的全局变量会得到 0 值，本地变量不会
    • 指针会得到 NULL
  • 只能用编译时刻已知的值为全局变量进行初始化
  • 它们的初始化发生在 main 函数之前

静态本地变量

  • 在本地变量定义时加上 static 修饰符就可以成为静态本地变量
  • 当函数离开时，静态本地变量会继续保持其值
  • 静态本地变量的初始化只会在第一次进入这个函数时做，以后进入函数时会保持上次离开时的值
  • 静态本地变量实际上是特殊的全局变量
  • 它们位于相同的内存区域
  • 静态本地变量具有全局的生存期和函数内部的作用域
  • 不要使用全局变量来在函数间传递参数结果
  • 尽量避免使用全局变量
  • 使用全局变量和静态本地变量是线程不安全的

# 函数的一些高级特性

# 函数的默认参数

在 C++ 中，函数的形参列表中的形参是可以有默认值的。

语法： 返回值类型 函数名(参数 = 默认值) {}

注意事项：

  1. 如果某个位置已经有了默认参数，那么从这个参数以后，都必须有默认值；
  2. 函数的声明和实现只能由一个写默认参数

# 函数的重载

意义：函数名可以相同，提高复用性

函数重载需要满足的条件：

  • 同一个作用域下
  • 函数名相同
  • 函数参数类型不同或者个数不同或者顺序不同
  • 函数的返回值不能作为函数重载的条件

函数重载的注意事项：

  1. 引用作为函数重载的条件

  2. 函数重载碰到函数的默认参数
